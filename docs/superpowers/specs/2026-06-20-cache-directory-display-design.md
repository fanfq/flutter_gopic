# 缓存目录展示

## 目标

在 Cloud 配置页的“图床缓存”卡片中，除缓存文件总数和占用空间外，展示当前本地缓存的实际存放目录。

## 设计

- `HistoryService` 继续作为缓存位置的唯一来源；界面不自行拼接路径。
- `CacheSummary` 增加缓存目录路径，使文件数、容量和路径来自同一次缓存统计。
- 配置页在统计完成后依次显示文件数、容量与“缓存目录：<path>”。路径文本可自动换行。
- 清除操作与现有逻辑保持不变，仍清除所展示的同一目录；上传历史和云端文件不受影响。
