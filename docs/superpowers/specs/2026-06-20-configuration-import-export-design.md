# GoPic 配置导入与导出设计

## 目标

为换机使用提供完整的云服务配置迁移能力。用户可导出本机配置 JSON，并在另一台设备导入；导入采用合并而非覆盖。

## 范围

- 在 macOS 风格侧边栏增加“配置”入口。
- 导出所有云配置、当前激活配置和压缩配置。
- 通过系统文件选择器导入和导出 JSON 文件。
- 导入后立即持久化，并刷新依赖配置的界面。

不导出上传历史、本地缩略图缓存或云端文件。

## 文件格式

导出文件使用带版本号的 JSON 包装：

```json
{
  "version": 1,
  "exportedAt": "2026-06-20T00:00:00.000Z",
  "cloud": {
    "profiles": [],
    "activeProfileId": "...",
    "compression": {}
  }
}
```

`cloud` 的内容复用现有 `CloudModel.toMap()` 结构，以保证所有云账号字段、启用状态、当前图床和压缩参数都能被完整恢复。文件会包含 Access Key 与 Secret Access Key 明文；配置页必须在导出操作旁清晰提示用户妥善保管。

## 导入与合并规则

1. 选择文件后先解析 JSON，并验证包装版本、`cloud` 对象及配置列表的基本结构。
2. 配置以稳定 `id` 为唯一身份：本机存在相同 ID 时，使用导入项更新；不存在时新增。
3. 导入的压缩配置覆盖本机压缩配置。
4. 若导入文件的 `activeProfileId` 在合并后的列表中可选，则设为当前图床；否则保留模型的现有可用选择逻辑。
5. 导入成功后一次性写入 `shared_preferences`；任意解析或校验失败都不得改变内存或本地持久化的配置。

完成提示需展示新增和更新数量。格式错误、版本不支持或无法读取文件时显示可理解的错误反馈。

## 界面

配置页沿用当前的页面标题、留白和内容卡片：

- “导出配置”卡片：说明迁移范围与明文密钥提示，主操作为“导出配置”。
- “导入配置”卡片：说明导入为合并操作，主操作为“选择配置文件”。

系统文件选择器负责选择保存位置或导入文件；默认导出名为带日期的 `gopic-config-YYYY-MM-DD.json`。

## 测试

- 配置备份可包含完整云配置、压缩配置和当前图床。
- 同 ID 导入更新现有配置，不新增副本。
- 新 ID 导入新增配置。
- 无效数据或不支持的版本被拒绝，且不改变现有模型。
- 服务层导入成功后持久化结果。
